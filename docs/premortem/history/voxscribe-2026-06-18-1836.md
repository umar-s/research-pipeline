### H-001: medium+CUDA/fp16 default → OOM-краш на 4GB GPU на первом прогоне
- **Угол зрения:** Допущения, Граничные случаи
- **Найдена:** 2026-06-18 (запуск 1)
- **Важность:** высокая
- **Уверенность:** подкреплено контекстом
- **Статус:** ПРИНЯТО
- **Режим:** полный
- **Описание:**
  [verified] whisper default --device cuda --fp16 True; машина RTX A400 4GB, ~2GB free; medium ~5GB VRAM → torch.cuda.OutOfMemoryError ДО транскрипции. Спека предполагала «CPU медленно», реальность — hard OOM-краш. CPU-fallback whisper сам не делает.
- **Решение:** transcribe.sh авто-детектит свободный VRAM (nvidia-smi memory.free) vs потребность модели; не влезает или нет GPU → --device cpu --fp16 False; дефолт-модель small (а не medium); всё переопределяемо --device/--model
  - Зачем выбрали: Горизонт = первый реальный прогон; незакреплённый device = гарантированный краш, а не деградация скорости
  - Первые шаги (prevent):
    1. §4.5: nvidia-smi free vs таблица VRAM (tiny/base~1,small~2,medium~5,large~10)
    2. дефолт model=small
    3. на cpu форс --fp16 False
    4. smoke: дефолтный прогон не падает OOM
  - Исполнитель: агент
  - Сигнал успеха (detect): дефолтный прогон отдаёт текст, не torch.cuda.OutOfMemoryError
  - Стоп-условие: OOM/traceback на первом прогоне с дефолтами
  - Если уже случилось (limit damage): явный --device cpu вручную

### H-002: whisper exit 0 = ложный успех: пустой/галлюцинированный txt на не-речи
- **Угол зрения:** Исполнитель
- **Найдена:** 2026-06-18 (запуск 1)
- **Важность:** высокая
- **Уверенность:** подкреплено контекстом
- **Статус:** ПРИНЯТО
- **Режим:** полный
- **Описание:**
  [verified] на тишине/синусе whisper exit 0, stderr чист, пишет 0-байтный или галлюцинированный (25-байт) .txt. Скрипт рапортует успех по наличию файла, не по содержанию. set -euo pipefail не ловит. Паттерн BA-91 false-green.
- **Решение:** output-sanity guard после whisper: .txt пуст/whitespace ИЛИ .json без сегментов/высокий no_speech_prob → явное предупреждение + exit 4, не молчаливый успех
  - Зачем выбрали: Молчаливая пустая/ложная расшифровка хуже падения — подрывает доверие с 1-го прогона, травит downstream данными-призраками
  - Первые шаги (prevent):
    1. §4.8: проверка непустоты .txt + наличия сегментов в .json
    2. exit 4 на пустой транскрипции
    3. печать реального текста, не только путей
    4. smoke: тишина → guard срабатывает
  - Исполнитель: агент
  - Сигнал успеха (detect): на нормальном аудио .txt непустой и осмысленный; на тишине — явное предупреждение
  - Стоп-условие: exit 0 при пустом/мусорном .txt
  - Если уже случилось (limit damage): пере-прогон с верным языком/моделью

  - Стоп-условие: уверенный русский на явно не-русском входе

### H-005: жёсткая привязка к openai-whisper без preflight → тихая поломка на иной машине
- **Угол зрения:** Будущий поддерживающий
- **Найдена:** 2026-06-18 (запуск 1)
- **Важность:** высокая
- **Уверенность:** подкреплено контекстом
- **Статус:** ПРИНЯТО
- **Режим:** полный
- **Описание:**
  transcribe.sh зовёт whisper как данность. На macOS/публичном MIT-юзере типична whisper.cpp (иной бинарь/флаги) или faster-whisper или нет GPU. Скрипт не проверяет бинарь, не даёт actionable-ошибку, флаги openai-whisper дрейфуют между версиями → через полгода/на другой машине молча ломается, чинить некому.
- **Решение:** preflight command -v whisper ffmpeg ffprobe с actionable-ошибкой; зафиксировать поддерживаемое окружение (Linux+openai-whisper) в options.md; macOS/whisper.cpp — known limitation v1, loud fail не silent
  - Зачем выбрали: «Ставится через /plugin marketplace» создаёт ожидание «работает у всех», а MIT-репо отдаёт скрипт людям без ~/.local/bin/whisper
  - Первые шаги (prevent):
    1. §4.2: preflight deps + exit 2 с инструкцией установки
    2. options.md: явное поддерживаемое окружение + known limitations
    3. версионная заметка про дрейф флагов whisper
  - Исполнитель: агент
  - Сигнал успеха (detect): отсутствие зависимости → понятная ошибка установки, не traceback
  - Стоп-условие: тихий traceback вместо «установи whisper»
  - Если уже случилось (limit damage): документированный fallback-движок
